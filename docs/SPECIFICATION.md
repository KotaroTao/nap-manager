# デンタルNAPマネージャー 仕様書

## 1. プロジェクト概要

### 1.1 プロジェクト名
**デンタルNAPマネージャー**（Dental NAP Manager）

### 1.2 目的
歯科医院のNAP（Name, Address, Phone）情報をWEB上で統一管理し、確認すべきサイトの一覧管理、確認リマインダー、修正依頼テンプレート生成を通じて、MEO（Map Engine Optimization）対策を効率化するWEBツール。

**本システムの特徴**
- スクレイピングは行わず、法的リスクを回避
- ユーザーが手動でサイトを確認し、結果を記録
- 確認すべきサイト一覧と各サイトへのリンクを提供
- 修正依頼テンプレートを自動生成し、対応を効率化

### 1.3 NAP統一の重要性
- Googleマップ等の検索エンジンは、NAP情報の一貫性を信頼性の指標として評価
- 表記揺れ（例：「歯科」vs「デンタル」、番地表記の違い等）があるとSEO/MEO評価が下がる
- 定期的な確認と是正作業の管理が重要

### 1.4 主要ユーザー
- 歯科医院経営者・スタッフ
- 歯科医院のWEBマーケティング担当者
- MEO対策を請け負う代理店

## 3. 機能詳細

### 3.1 認証システム

#### 3.1.1 新規登録
- **登録方法**: メールアドレス認証
- **必要情報**:
  - メールアドレス（必須、ユニーク）
  - パスワード（必須、8文字以上、英数字記号混在推奨）
  - 医院名（クライアントの場合、必須）
  - 氏名（必須）
  - 電話番号（任意）
- **フロー**:
  1. 登録フォーム入力
  2. 確認メール送信（認証リンク付き）
  3. リンククリックで本登録完了
  4. 14日間無料トライアル開始

#### 3.1.2 ログイン
- **方式**: メールアドレス + パスワード
- **セッション管理**:
  - セッション有効期間: 24時間
  - 「ログイン状態を維持」選択時: 30日間
  - 同時ログイン: 許可（複数デバイス対応）

#### 3.1.3 パスワードリセット
- **フロー**:
  1. パスワードリセット申請（メールアドレス入力）
  2. リセットリンク付きメール送信
  3. リンク有効期限: 24時間
  4. 新パスワード設定

#### 3.1.4 二要素認証（将来拡張）
- TOTP（Google Authenticator等）対応を将来的に検討

### 3.4 NAP確認管理

#### 3.4.1 確認ワークフロー

1. **確認リマインダー受信**: 確認推奨頻度に基づきメール通知
2. **サイトを開く**: システムから該当サイトへのリンクをクリック
3. **NAP情報を目視確認**: ユーザーがサイト上で医院情報を確認
4. **結果を記録**: システムに確認結果を入力
5. **必要に応じて修正依頼**: 不一致の場合、テンプレートを生成

#### 3.4.2 ステータス定義（ユーザーが選択）

| ステータス | 説明 | 表示色 |
|------------|------|--------|
| **一致** | 正式NAPと一致している | 緑 |
| **不一致** | NAP情報に差異がある（修正必要） | 赤 |
| **未掲載** | 医院情報がサイトに掲載されていない | オレンジ |
| **未確認** | まだ確認していない | グレー |

#### 3.4.3 確認結果の入力項目

**ユーザーが入力する情報**
- ステータス（一致/不一致/未掲載/未確認）
- サイトで確認した医院名（任意）
- サイトで確認した住所（任意）
- サイトで確認した電話番号（任意）
- メモ（任意）
- 確認日時（自動記録）

#### 3.4.4 ダッシュボード表示

**サマリー**
- 不一致件数のバナー表示（赤背景）
- 確認期限が近いサイトのバッジ表示（黄背景）
- サイト別ステータス一覧

**確認推奨リスト**
- 前回確認から推奨頻度を超過したサイト一覧
- 優先度順にソート

#### 3.4.5 通知設定（クライアント個別）
- 確認リマインダー: ON/OFF（推奨頻度に基づく）
- 週次サマリー: ON/OFF
- 月次レポート: ON/OFF
- 通知先メールアドレス（複数登録可）

### 3.5 修正依頼機能

#### 3.5.1 修正依頼テンプレート
サイト種別ごとに最適化されたテンプレートを自動生成。
確認時に入力した情報と正式NAP情報を自動で差し込み。

**テンプレート例（口コミサイト向け）**
```

○○サイト ご担当者様

お世話になっております。
○○歯科クリニックの担当△△と申します。

貴サイトに掲載いただいている当院の情報につきまして、
下記の通り訂正をお願いしたくご連絡いたしました。

■ 現在の掲載情報（確認日: YYYY/MM/DD）
医院名: [確認時に入力した医院名]
住所: [確認時に入力した住所]
電話番号: [確認時に入力した電話番号]

■ 正しい情報
医院名: [正式医院名]
住所: [正式住所]
電話番号: [正式電話番号]

お手数をおかけいたしますが、ご対応のほど
よろしくお願いいたします。

○○歯科クリニック
担当: △△
連絡先: XXX-XXXX-XXXX
```

#### 3.5.2 対応状況トラッキング

| ステータス | 説明 |
|------------|------|
| 未対応 | 修正依頼未送信 |
| 依頼中 | 修正依頼送信済み、返答待ち |
| 対応中 | サイト側で修正作業中 |
| 完了 | 修正確認済み |
| 対応不可 | サイト側で修正不可と回答 |

#### 3.5.3 リマインダー
- 依頼後7日間返答がない場合、リマインダー通知
- 依頼後14日間変化がない場合、再送信提案

### 3.6 Stripe課金システム

#### 3.6.1 プラン構成

| プラン | 月額（税抜） | 医院数 | マスタサイト | 手動追加サイト | リマインダー |
|--------|-------------|--------|--------------|----------------|--------------|
| **ライト** | ¥1,980 | 1医院 | 上位10サイト | 最大10件 | 月次 |
| **スタンダード** | ¥3,980 | 3医院 | 全サイト | 最大30件/医院 | 週次 |
| **エンタープライズ** | ¥9,800 | 10医院 | 全サイト | 無制限 | カスタム |

#### 3.6.2 無料トライアル
- 期間: 14日間
- 機能: スタンダードプラン相当
- クレジットカード登録: 不要（トライアル開始時）
- トライアル終了3日前にメール通知
- トライアル終了後、プラン選択画面へ誘導

#### 3.6.3 Stripe機能
**Customer Portal**
- プラン変更
- 支払い方法変更
- 請求書ダウンロード
- 解約手続き

**Webhook対応イベント**
- checkout.session.completed（新規契約）
- customer.subscription.updated（プラン変更）
- customer.subscription.deleted（解約）
- invoice.payment_failed（支払い失敗）
- invoice.payment_succeeded（支払い成功）

#### 3.6.4 支払い失敗時の対応
- 即座にメール通知
- 3日間の猶予期間
- 猶予期間後、アカウント一時停止（読み取り専用）
- 14日間支払いがない場合、データ保持のみ（機能停止）

## 4. 非機能要件

### 4.1 パフォーマンス
- ページロード時間: 3秒以内
- API レスポンス時間: 500ms以内（95パーセンタイル）
- 同時接続数: 1000ユーザー以上対応

### 4.2 可用性
- 稼働率: 99.5%以上
- 計画メンテナンス: 月1回、深夜帯（2:00-5:00）

### 4.3 セキュリティ
- HTTPS必須
- パスワードハッシュ化（bcrypt）
- SQLインジェクション対策
- XSS対策
- CSRF対策
- レートリミッティング

### 4.4 データ保持
- アクティブアカウント: 無期限
- 解約アカウント: 30日後にデータ削除
- ログ: 90日間保持

### 4.5 バックアップ
- データベース: 日次バックアップ
- 保持期間: 30日間
- リストア: 4時間以内
